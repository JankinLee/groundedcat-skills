# 剧情发展健康度评估清单

评估故事整体进展节奏、悬念管理、人物弧线和结构完整性，确保剧情可持续推进。

## 0. 章节信息回填检查

每章完成后，必须执行以下信息回填动作，确保项目文档与故事状态同步：

### 章节摘要回填
- [ ] **本章摘要已更新**：在 `02-项目管理.md` 的“章节摘要”部分添加本章概要
- [ ] **人物状态变化记录**：在 `02-项目管理.md` 的“角色当前状态”表中更新相关角色
- [ ] **伏笔台账更新**：在 `02-项目管理.md` 的“伏笔台账”中标记已回收伏笔，录入新伏笔
- [ ] **进度数据同步**：在 `02-项目管理.md` 的“字数与进度”中更新累计字数和完成章节数

### 按需文档更新
- [ ] **人物档案更新**：如本章有新角色出场、角色性格重大成长、背景揭示或关系质变，更新 `01-人物档案.md`
- [ ] **世界观补充**：如本章引入新设定、地点或规则，更新 `03-世界观与时间线.md`（如有）
- [ ] **章节规划调整**：如本章导致原剧情规划变化，在 `04-章节规划.md` 中标注变更说明

### 回填质量检查
- [ ] **信息一致性**：回填内容与章节正文无矛盾
- [ ] **术语统一**：名称、概念与已有设定保持一致
- [ ] **时效性**：回填在章节交付后立即完成

> **执行时机**：每章创作完成后，在交付前或交付后立即执行。回填结果作为健康度评估的基础数据。

## 1. 节奏与进展评估

### 章节推进效率
- [ ] **本章有实质进展**：不是过渡章、说明章或重复章
- [ ] **信息释放节奏**：重要信息分散释放，不是大段信息轰炸
- [ ] **紧张与舒缓交替**：高强度场景后有适当缓冲，维持读者注意力

### 故事阶段判断
- [ ] **当前处于正确阶段**：根据采用的结构模板（三幕、五幕、英雄之旅等），本章功能符合当前位置要求
- [ ] **阶段转换标志**：如有阶段转换（如从第一幕进入第二幕），转换点是否清晰合理
- [ ] **进度比例适当**：当前章节数占总计划章节数的比例与故事进展程度匹配

## 2. 悬念管理评估

### 伏笔平衡
- [ ] **新旧伏笔比例**：新开伏笔数量 ≤ 回收伏笔数量 × 2
- [ ] **伏笔生命周期**：长期伏笔（>10章）有阶段性揭示，短期伏笔（<3章）及时回收
- [ ] **伏笔密度**：每章新增伏笔不超过3个（悬疑题材可适当增加）

### 悬念强度梯度
- [ ] **主悬念持续牵引**：每3-5章至少有一次主悬念推进
- [ ] **次悬念交错**：不同线索的悬念高潮错开，避免同时爆发
- [ ] **悬念类型多样**：信息悬念、关系悬念、安全悬念、身份悬念等混合使用

## 3. 人物弧线评估

### 主角发展
- [ ] **内在成长可见**：主角的价值观、技能、人际关系有明显变化
- [ ] **缺陷与克服**：主角的致命缺陷在剧情压力下暴露并有克服迹象
- [ ] **欲望升级**：主角的欲望随剧情进展而深化或转变

### 配角发展
- [ ] **配角有独立弧线**：重要配角有自己的目标、阻碍和变化
- [ ] **关系网演化**：角色间关系不是静态的，随事件变化
- [ ] **角色功能不重叠**：不同角色在剧情中承担不同功能，避免冗余

## 4. 冲突管理评估

### 冲突层次
- [ ] **外部冲突持续**：主角与 antagonist 的对抗有进展
- [ ] **内部冲突显现**：主角内心矛盾在关键时刻影响决策
- [ ] **关系冲突发展**：主角与盟友、敌人的关系有变化

### 冲突升级
- [ ] **代价递增**：冲突的代价随剧情进展而增加
- [ ] **风险提高**：失败的风险越来越大
- [ ] **选项减少**：主角的选择空间逐渐收窄

## 5. 结构完整性评估

### 三幕结构检查（如采用）
| 阶段 | 应有功能 | 当前状态 |
|------|----------|----------|
| **第一幕（建制）** | 介绍主角、常态、激发事件 | 已完成 / 进行中 / 未开始 |
| **第二幕（对抗）** | 主角主动行动、中途点、灵魂黑夜 | 已完成 / 进行中 / 未开始 |
| **第三幕（解决）** | 高潮、结局、新常态 | 已完成 / 进行中 / 未开始 |

### 多线叙事平衡
- [ ] **主线清晰**：读者能明确识别主线剧情
- [ ] **支线服务主线**：支线剧情最终汇入主线或强化主题
- [ ] **切换频率适当**：多线切换不过于频繁，每线有足够篇幅展开

## 6. 读者体验评估

### 预期管理
- [ ] **承诺与兑现**：故事早期建立的承诺在后文有兑现
- [ ] **意外与合理**：转折既出人意料又在情理之中
- [ ] **情感波动**：章节间情感基调有变化，避免单调

### 认知负荷
- [ ] **新概念引入节奏**：每章引入的重要新概念不超过2个
- [ ] **信息回忆提示**：重要旧信息在需要时有适当提示
- [ ] **术语一致性**：同一概念始终使用相同术语

## 7. 项目健康度指标

| 指标 | 当前值 | 健康范围 | 状态 |

## 8. 问题诊断与修复建议

### 常见问题模式

#### 节奏过慢
- **症状**：连续3章无实质进展，角色在原地打转
- **修复**：
  1. 合并或删除过渡章
  2. 增加时间压力（倒计时、最后期限）
  3. 引入新冲突或转折

#### 伏笔堆积
- **症状**：待回收伏笔数量 > 已回收伏笔数量 × 3
- **修复**：
  1. 规划集中回收章节
  2. 合并相关伏笔
  3. 放弃次要伏笔

#### 角色停滞
- **症状**：主要角色连续5章无变化
- **修复**：
  1. 给予该角色独立抉择场景
  2. 揭示该角色隐藏背景
  3. 改变该角色与其他角色的关系

#### 冲突疲软
- **症状**：冲突代价未升级，风险未提高
- **修复**：
  1. 提高失败后果的严重性
  2. 引入新的 antagonistic 力量
  3. 让主角失去重要资源或盟友

## 9. 评估周期与介入点

### 定期评估
- **每3章**：全面健康度评估（使用评分卡）
- **每10章**：结构完整性深度检查
- **关键节点**：第一幕结束、中点、灵魂黑夜、高潮前

### 介入阈值
| 健康度总分 | 状态 | 建议行动 |
|------------|------|----------|
| ≥85 | 优秀 | 保持当前方向，可适当增加复杂度 |
| 70-84 | 健康 | 正常推进，关注弱项维度 |
| 60-69 | 亚健康 | 制定针对性改善计划 |
| <60 | 不健康 | 暂停创作，进行结构调整 |

### 剧情推进或变更需求评估

每3章全面评估后，需额外评估剧情推进效率与变更需求，确保故事方向与创作目标一致：

#### 推进效率评估
- [ ] **剧情推进速度**：当前推进速度是否与计划相符（过快/过慢/适中）
- [ ] **章节功能实现**：已完成章节是否有效实现预设功能（建立、发展、转折、高潮等）
- [ ] **资源利用效率**：已引入的设定、角色、伏笔是否得到充分利用

#### 变更需求识别
- [ ] **意外机会发现**：创作过程中是否发现比原计划更好的伏笔、转折或角色发展机会
- [ ] **剧情扩展潜力**：当前情节线是否有扩展空间（利大于弊）
- [ ] **方向调整信号**：角色自主行为、读者反馈或作者直觉是否暗示需要调整方向

#### 变更决策框架
| 变更类型 | 触发条件 | 建议行动 |
|----------|----------|----------|
| **微调优化** | 局部情节可增强，不影响主线 | 调整细节，不修改大纲 |
| **扩展补充** | 发现优质扩展点，需增加篇幅 | 在原章节规划.md中添加分支章节 |
| **方向调整** | 角色/情节自然发展到更好方向 | 更新章节规划核心路径，调整后续规划 |
| **结构重构** | 整体结构存在问题，需重新规划 | 暂停创作，重新设计章节规划 |

#### 用户介入流程
1. **呈现评估结果**：展示推进效率评分与变更需求识别
2. **提出变更建议**：基于上述框架，提供具体变更选项
3. **用户决策**：用户选择变更类型与具体方案
4. **文档更新**：根据决策更新 `00-大纲.md`、`01-人物档案.md`、`02-项目管理.md` 等相关文档
5. **记录决策**：在 `02-项目管理.md` 的“自检流程配置”中记录变更原因与内容

> **执行时机**：每3章全面健康度评估后自动触发。变更决策需经用户明确确认后方可执行。

## 10. 与连载模式介入环节集成

此评估清单应在连载模式的“作者介入环节”中使用，集成章节信息回填、健康度评估与剧情变更反馈：

1. **生成当前故事状态简报**：
   - 包含健康度评分卡（节奏、悬念、人物、冲突、结构、体验）
   - 包含章节信息回填状态（摘要、人物状态、伏笔台账更新情况）
   - 包含剧情推进效率评估与变更需求识别结果

2. **根据评估结果提示问题**：
   - 健康度总分低于阈值时，提示风险维度
   - 章节信息回填缺失时，提示补全
   - 发现优质变更机会时，提示扩展或调整可能性

3. **提供基于评估结果的介入选项**：
   - **健康度修复选项**：针对低分维度的具体改善方案
   - **信息回填补全选项**：缺失回填项的快速补全
   - **剧情变更选项**：基于“剧情推进或变更需求评估”的变更建议（微调、扩展、方向调整、结构重构）

4. **执行用户决策与文档更新**：
   - 记录用户选择的介入方案
   - 更新相关文档（大纲、人物档案、项目管理文件）
   - 在 `02-项目管理.md` 的“自检流程配置”中记录评估结果与变更决策

5. **跟踪趋势与持续优化**：
   - 记录每次评估结果，跟踪健康度变化趋势
   - 分析变更决策的效果，优化后续评估准确性
   - 根据连载进度调整评估频率与重点

> **使用提示**：在连载模式中，每完成设定章数（默认3章）后运行此评估，且**必须**包含章节信息回填检查与剧情变更需求评估。在策划模式中，用于大纲的结构健康度检查。在完稿模式中，终稿前需全本评估。
