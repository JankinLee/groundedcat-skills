# 质量检查清单

交付任何章节或整书前，先确认当前模式，再按对应清单检查。

## 0. 模式确认

- [ ] 当前是 `策划模式 / 试写模式 / 连载模式 / 完稿模式`
- [ ] 目标单章字数已锁定
- [ ] 本次交付目标清晰：大纲 / 样章 / 单章 / 连续章节 / 全书收尾

## 1. 章节基础

- [ ] **章节有明确功能**
  - 推剧情 / 推关系 / 推人物弧线 / 回收伏笔，至少命中其一
- [ ] **字数符合项目设定**
  - 轻连载：2000-3000 字
  - 标准长章：2500-4000 字
  - 重度长章：3000-5000 字
- [ ] **本章不是可删章**
  - 删除后会造成信息缺口、关系断层或节奏问题
- [ ] **时间地点清晰**
  - 场景切换时读者不会迷路

## 2. 开头检查

### 2.1 章节类型判断（先确认）

- [ ] **确认本章核心功能类型**：
  - **冲突推进章**：推进主线冲突、解决危机、战斗对决（必须严格前20%原则）
  - **情节转折章**：重大揭示、身份暴露、关系突变（必须严格前20%原则）
  - **信息铺垫章**：线索收集、计划制定、技能学习（适度放宽）
  - **人物深化章**：角色背景揭示、心理变化、关系发展（可适当放宽）
  - **抒情氛围章**：情感抒发、环境渲染、哲学思考（**特例允许**后移效果）
  - **回忆插叙章**：闪回、往事回忆、背景补充（**特例允许**后移效果）
  - **过渡衔接章**：时间跳跃、地点转换、节奏调整（保持基础可读性）

### 2.2 基础检查（所有类型通用）

- [ ] **前3段内读者不会迷路**：时间、地点、人物状态清晰
- [ ] **避免所有"致命错误"**：天气描写、日常流程、回顾上章、缓慢铺垫、平淡对话、过度解释
- [ ] **与上一章衔接自然**：时间跳跃合理、地点转换明确、情绪过渡连贯
- [ ] **读者有继续阅读的理由**：至少一个明确的悬念、问题或情感钩子
- [ ] **开头服务本章核心功能**：不是无关的炫技或拖延

### 2.3 冲突推进章 & 情节转折章专项检查

- [ ] **前20%强度评估**（严格执行）：
  - 即时紧张感：读者在200字内感受到危机/冲突
  - 重大事件引入：前20%包含推动剧情的重要事件或信息
  - 情感冲击：读者产生强烈情绪（震惊、担忧、愤怒、好奇）
  - 继续阅读欲望：读者迫切想知道"接下来会发生什么"
- [ ] **冲突根源清晰**：读者理解为什么发生冲突
- [ ] **利害关系明确**：读者知道谁在冒险、赌注是什么
- [ ] **行动而非解释**：通过动作、对话、选择展示冲突，而非背景说明

### 2.4 慢热氛围型章节特例处理（抒情章、回忆章、人物章等）

**适用条件**：本章核心价值在于情感沉浸、氛围营造、人物深度而非情节推进

- [ ] **特例确认**：本章确实属于抒情、回忆、人物深化等慢热类型，而非节奏失误
- [ ] **效果后移合理性**：如果前20%较慢，需在\*\*前40%\*\*达到同等的情感/思想深度
- [ ] **补偿机制存在**：本章包含其他高价值元素（深度人物洞察、世界观揭示、哲学思考等）
- [ ] **氛围一致性**：开头建立的基调与本章整体氛围匹配
- [ ] **读者预期管理**：通过章节标题、上章结尾等方式暗示本章节奏

### 2.5 快速诊断与修复建议

**如发现问题，参考以下修复方向**：

- **开头平淡无味**：删除前3段，从第一个冲突点开始；添加倒计时、威胁、异常元素；用对话替代描述
- **开头与本章脱节**：重写开头，直接进入本章核心；调整开头，建立与后续内容的关联
- **开头过于急促**：添加1-2句背景说明；通过角色内心活动建立上下文；使用闪回或对话暗示前因
- **慢热但无补偿**：提升后续部分的情感或思想深度；增加本章的信息价值；如无法修复，考虑删除或合并本章

**特例章节修复**：

- 如确认为慢热氛围型但开头质量不足，可：增强语言美感、意象独特性；提前暗示本章将带来的情感或思想回报；缩短铺垫长度（效果后移点从40%提前到30%）；添加微悬念维持读者兴趣

### 如果这是首章

- [ ] 主角已经立住，不是模糊旁观者
- [ ] 这本书的核心卖点已经显形
- [ ] 首章结尾抛出了具体主钩子

## 3. 内容推进

- [ ] **本章有不可逆变化**
  - 信息被揭示、关系被改变、局势被升级、人物做出代价性选择
- [ ] **冲突持续存在**
  - 不是只聊天、只解释、只回忆
- [ ] **有转折**
  - 至少有一次预期被改变或认知被修正
- [ ] **因果成立**
  - 事件不是靠巧合硬推

## 4. 人物与对白

- [ ] **人物行为符合既有设定**
  - 若反常，文本内给出原因
- [ ] **主要角色声音可区分**
  - 用词、语气、句长、态度存在差异
- [ ] **每段对白有功能**
  - 推动情节 / 暴露关系 / 制造冲突 / 放出信息
- [ ] **潜台词成立**
  - 角色不会把所有心事都直白说完
- [ ] **对话与世界观设定契合**
  - 对话用语与世界观设定相符（如科幻世界尽量不用文言文）
  - 特殊设定（如术语）的使用应符合角色认知水平和场景需求

## 5. 叙事表达

- [ ] **展示大于讲述**
  - 少用抽象情绪总结，多用动作、细节、反应和选择
- [ ] **细节服务剧情**
  - 不是为了凑字数而堆环境描写
- [ ] **句式有变化**
  - 长短句交替，避免整段平均长度
- [ ] **段落可读性好**
  - 动作场景不堆成文字墙

## 6. AI 味清理

- [ ] **不随意打破第四面墙？**（严格执行）
- [ ] **基于现有设定而非模型了解创作？**（严格执行）
- [ ] **人物描写是否基于作品已有设定的人物和设定，除非人物设定不完备？**（严格执行）
- [ ] **不要出现Markdown语法、正则表达式等非正常创作内容**（严格执行）
- [ ] **删掉空泛修饰词**
  - 如“复杂的情绪”“深邃的目光”“莫名的感觉”这类可替换空话
- [ ] **删掉机械排比和成语堆砌**
  - 不用四字词串成整段情绪
- [ ] **减少解释腔**
  - 少用“这意味着”“然而”“与此同时”“某种程度上”这类说明式连接
- [ ] **保留人物口气**
  - 角色对话不像统一的作者口吻
- [ ] **避免元叙事表达**
  - 不在正文中直接提及“伏笔”、”转折“、”高潮“、”矛盾“、“设定”等创作元叙事概念，通过情节自然暗示与印证

## 7. 悬念与连贯性

- [ ] **结尾有钩子**
  - 回应、升级或偏转至少一条悬念线
- [ ] **旧伏笔有推进**
  - 不是只开新坑不填旧坑
- [ ] **世界规则未被破坏**
  - 特别是悬疑、奇幻、科幻中的规则系统
- [ ] **人物状态已更新**
  - 位置、情绪、关系、公开目标、隐藏动机有记录

## 8. 类型特定补充

### 悬疑 / 推理

- [ ] 线索公平
- [ ] 红鲱鱼不过度作弊
- [ ] 真相能回溯自洽

### 言情 / 情感

- [ ] 关系温差有推进
- [ ] 吸引与阻碍同时存在
- [ ] 情感节点不是空喊“喜欢”

### 奇幻 / 科幻

- [ ] 规则有代价
- [ ] 世界信息是嵌入剧情而非讲义
- [ ] 新设定不会临时救场

### 动作 / 惊悚

- [ ] 空间方位清晰
- [ ] 危机层层升级
- [ ] 紧张和缓冲节奏交替

## 9. 完稿模式额外检查

- [ ] 主线冲突已收束
- [ ] 主角弧线有兑现
- [ ] 关键配角去向明确
- [ ] 重大伏笔已回收或明确保留到续作
- [ ] 结局情绪与题材匹配
- [ ] 没有为了续作接口牺牲本书完整性
- [ ] 终章和尾声分工清楚，没有重复收尾
