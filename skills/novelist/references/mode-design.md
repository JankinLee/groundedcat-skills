# 模式执行准则

定义不同模式的执行规则。

## 模式定义

1. `策划模式`：只做题材定位、故事引擎、大纲、人物、世界观、伏笔台账、章节规划
2. `试写模式`：策划完成后，只写首章或样章
3. `连载模式`：按章推进，每次交付 1 章或少量连续章节
4. `完稿模式`：只有在用户明确要求整本初稿时，才连续写完整部长篇

## 交互内容定义

**状态简报**: 当前故事状态的详细报告，包括主线进度、人物状态、伏笔台账、结构进度等。

### 状态简报示例

```
当前故事状态（截至第 8 章）

【主线进度】
- 主线冲突：主角调查旧案，已发现关键证人 X。
- 当前阻碍：证人 X 突然失踪，线索中断。
- 主悬念状态：仍待回收（计划第 12 章揭示）。

【人物状态】
- 主角：情绪焦虑，开始不信任搭档。
- 反派：尚未正面出场，但已通过间接手段施压。
- 关键配角：搭档行为可疑，读者已产生怀疑。

【伏笔台账】
- 待回收伏笔：5 条（其中 2 条已超过建议回收章节）
- 新增伏笔（最近 3 章）：1 条

【结构进度】
- 采用结构：悬疑结构（10 章）
- 当前位置：第 8 章，即将进入第三幕（真相揭示）
```

**介入方式**: 提供用户选择的介入方式，包括调整方向、补充设定、指定重点等。

### 介入方式示例

```
请选择你的介入方式：

1. 调整主线方向
   - 例如：增加新线索、改变嫌疑人、加入第三方势力

2. 补充剧情设定
   - 例如：添加新配角、补充世界观细节、设定新规则

3. 指定下一阶段重点
   - 例如：接下来 3 章聚焦主角与搭档的关系变化
   - 例如：尽快回收“匿名信”伏笔

4. 修改已有设定
   - 例如：修改某个人物动机、调整伏笔回收方式

5. 继续按原计划进行（无调整）
```

## 连载模式执行规则

1. 默认按章节顺序逐章创作。
2. 在以下情况触发“作者介入环节”：
   - 每完成 N 章（N 由用户设定，默认 3 章）, 维护在 `02-项目管理.md`中的`连载介入阈值`
   - 到达结构模板的关键节点（如中点、高潮前）
   - 用户主动输入“暂停”或“调整剧情”
3. 介入环节执行：
   - 生成当前故事`状态简报`
   - 提供`介入方式`,询问用户是否需要调整方向、补充设定或指定重点
   - 根据用户输入更新相关文档
     - 若调整主线，更新`04-章节规划.md`。
     - 若补充设定，更新`01-人物档案.md`或`02-项目管理.md`或`03-世界观与时间线.md`中。
     - 若指定重点，在后续章节的`本章任务卡`中强化该方向。
   - 用户确认“继续”后，恢复创作
