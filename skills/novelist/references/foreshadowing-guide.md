# 伏笔管理指南

负责维护`02-项目管理.md`中的伏笔台账。

## 长线伏笔台账

| 伏笔ID | 类型/事件           | 首次出现 | 当前状态 | 计划回收章节 | 实际回收章节 | 关联人物 | 关联悬念 |
| ------ | ------------------- | -------- | -------- | ------------ | ------------ | -------- | -------- |
|        | 主线悬念 / 支线伏笔 |          | 待回收   |              |              |          |          |

### 字段说明

**伏笔ID**：唯一标识，便于引用。例如：F-001、F-002等。
**类型**：物品/信息/事件/关系/规则。例如：物品伏笔、信息伏笔、事件伏笔、关系伏笔、规则伏笔等。
**首次出现**：伏笔首次出现的章节号+具体位置（可选）。例如：第3章、第5章第10行等。
**当前状态**：待回收 / 已回收 / 已废弃。例如：待回收、已回收、已废弃等。
**计划回收章节**：预设的回收章节（可由 AI 推荐）。例如：第10章、第15章等。
**实际回收章节**：最终回收的章节。例如：第10章、第15章等。
**关联人物**：涉及的关键人物。例如：主角、反派、配角等。
**关联悬念**：若属于某个主线/支线悬念，可关联。

### 伏笔回收检查

1. 从 02-项目管理.md 读取所有“待回收”的伏笔。
2. 根据当前章节的剧情（从章节规划.md中获取本章功能），判断是否有伏笔自然适合在此章回收。
3. 若有匹配，构造推荐提示：

```
  当前章节正在处理 [情节内容]。以下伏笔可能适合在本章回收：
- [伏笔描述]（首次出现于第 X 章）
  建议：可以设计主角在 [场景] 中发现与这个伏笔相关的线索，或让某个人物提及此事。

```

4. 将推荐信息以注释形式输出给用户。

### 伏笔自动提取

1. 读取本章正文。
2. 分析章节正文，识别出所有可能是伏笔的元素(包括但不限于：新出现的物品、人物未解的秘密、反常的行为、未解释的事件、对话中的隐含信息等)
3. 将识别结果与现有伏笔台账对比，避免重复录入。若为新伏笔，则追加到 02-项目管理.md 的伏笔表中，并设置状态为“待回收”。

## 长期伏笔管理

对于长篇作品（100章以上），伏笔可能跨越数十甚至上百章，需要专门的管理策略。

### 阶段冗余检查

**检查周期**：
- **每10章**：快速检查所有待回收伏笔的状态
- **每30章**：全面评估伏笔平衡与回收计划
- **关键节点**：第一幕结束、中点、高潮前等结构节点

**检查内容**：
1. **伏笔密度评估**：
   - 待回收伏笔数量 / 已回收伏笔数量 ≤ 3（健康）
   - 待回收伏笔数量 / 总章节数 ≤ 0.5（平均每2章开1个新伏笔）
   - 长期伏笔（计划回收章节 > 当前章节 + 20）占比 ≤ 30%

2. **过期风险检查**：
   - 标记计划回收章节已过但未回收的伏笔
   - 标记首次出现章节距今超过30章仍未回收的伏笔
   - 标记关联人物已死亡或退场的伏笔

3. **效用评估**：
   - 评估每个待回收伏笔对主线/支线的贡献度
   - 识别可能重复或功能重叠的伏笔
   - 评估伏笔回收的预期读者影响

### 人工介入归档清理环节

**触发条件**：
1. 阶段冗余检查发现过期、重复或低效用伏笔
2. 剧情方向发生重大转变，原有伏笔失去意义
3. 作者主动发起清理（建议每50章至少一次）

**清理流程**：
```
1. 生成待清理伏笔清单
2. 提供每个伏笔的详细信息（首次出现、原计划、当前状态）
3. 提供清理选项：
   A. 标记为"已废弃"（明确放弃，可记录原因）
   B. 标记为"已归档"（暂时搁置，未来可能重启）
   C. 调整回收计划（修改计划回收章节）
   D. 合并到其他伏笔（功能重叠时）
   E. 保持原状（确认仍有价值）
4. 执行用户选择的清理操作
5. 更新伏笔台账，记录清理决策
```

**清理决策参考**：
- **废弃条件**：伏笔关联的情节线已被删除；伏笔核心设定与世界规则冲突；伏笔预期效果已被其他情节实现
- **归档条件**：伏笔仍有价值但当前剧情线无法容纳；需要更长时间酝酿；等待更适合的回收时机
- **调整条件**：原计划回收章节已过但仍想保留；剧情节奏变化需要提前或推迟回收

### 长期伏笔跟踪表

在 `02-项目管理.md` 中可添加长期伏笔跟踪表：

| 伏笔ID | 创建时间 | 计划回收时间 | 当前状态 | 最后检查时间 | 检查结果 | 下次检查时间 |
|--------|----------|--------------|----------|--------------|----------|--------------|
| F-001  | 第3章    | 第25章       | 待回收   | 第15章       | 正常     | 第25章       |
| F-002  | 第3章    | 第40章       | 待回收   | 第15章       | 风险：过期风险 | 第25章       |

**字段说明**：
- **创建时间**：伏笔首次出现的章节
- **计划回收时间**：预计回收的章节
- **当前状态**：待回收 / 已回收 / 已废弃 / 已归档
- **最后检查时间**：上次检查评估的章节
- **检查结果**：正常 / 风险：过期风险 / 风险：密度过高 / 建议清理
- **下次检查时间**：建议下次检查的章节

### 与连载模式介入环节集成

在连载模式的"作者介入环节"中，可加入伏笔管理选项：

```
当前伏笔状态（截至第N章）：
- 待回收伏笔：X个（其中长期伏笔Y个）
- 已回收伏笔：Z个（回收率：Z/(X+Z)%）
- 伏笔密度：待回收/已回收 = X/Z
- 高风险伏笔：A个（过期）、B个（重复）、C个（低效用）

请选择伏笔管理操作：
1. 查看高风险伏笔详情
2. 执行阶段冗余检查
3. 启动人工介入归档清理
4. 调整伏笔回收计划
5. 跳过伏笔管理，继续创作
```

### 最佳实践建议

1. **分层设计**：
   - 短期伏笔（3-10章回收）：维持即时悬念
   - 中期伏笔（10-30章回收）：支撑支线剧情
   - 长期伏笔（30+章回收）：服务主线核心悬念

2. **回收节奏控制**：
   - 避免连续多章只开新伏笔不回收旧伏笔
   - 高潮章节集中回收多个伏笔增强满足感
   - 保持伏笔回收的节奏变化（单独回收、组合回收、意外回收）

3. **读者记忆管理**：
   - 重要长期伏笔在前文中有规律地轻微提醒
   - 回收前1-3章进行适度铺垫但不泄露关键信息
   - 复杂伏笔系统可考虑提供"前情提要"式回顾
